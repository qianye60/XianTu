# 正式游戏对话·提示词拼装清单（发送顺序）

目的：规范每次“玩家→AI”的请求所携带的文件与变量，确保上下文完整、一致、可控。

发送顺序（上→下为优先级/靠前放置）：

1) 系统规则（强约束）
- `想法资料/提示词套件/00_系统总则.md`

2) 命令规范（速查，严格遵循）
- `想法资料/提示词套件/01_命令规范_精简卡.md`
- （可选补充）`想法资料/命令规范-增强版.md`（当模型需要更强可控性时附上）

3) 世界书（当前存档绑定的世界观与文风）
- 推荐路径：`想法资料/世界书/世界书_<存档ID>.md`（若未配置，则使用模板）
- 模板：`想法资料/世界书/世界书_模板.md`

4) 难度与合理性审查（最高权限，防不公/不真实）
- `想法资料/提示词套件/03_合理性审查_难度策略.md`

5) 叙事风格与语气（一致性）
- `想法资料/提示词套件/04_叙事风格.md`

6) 运行时注入（由后端注入的动态片段，非文件）
- 当前关键变量快照（JSON 片段）：
  - `character.saveData.角色基础信息.*`
  - `character.saveData.玩家角色状态.*`
  - `character.saveData.记忆.(长期|中期|短期)`（可裁剪）
  - `character.saveData.任务.*`（进行中的任务）
  - `character.saveData.世界信息.*`（必要摘要）
  - `character.saveData.时间.{当前,时间轴[-N..]}`
- 最近对话摘要：最近 N 轮对话（对玩家可见的 text）

7) 用户本轮输入（玩家操作/对话）

说明：
- 文件 1–5 为“静态提示词”，除非编辑文档，否则不会频繁变化；6 为“动态注入”，每轮刷新；7 为本轮输入。
- 若 token 受限：优先保留（1）系统总则、（2）命令速查、（6）关键变量与摘要、（7）用户输入；世界书可换成“当前场景摘要”。
